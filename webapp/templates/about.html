How it works

Five gates. One verdict.

Most fake-news projects throw a single ML model at a headline and call it a day. {{ brand }} takes the opposite approach: a real headline should pass five independent sanity checks, each catching a different failure mode.

Why five checks?

Each gate is intentionally narrow and explainable. Together they cover the most common signals of a fabricated headline:

  1. Spelling. Real journalism is edited; fake stories often contain obvious typos.
  2. Clickbait. A trained classifier rejects attention-bait phrasing.
  3. Subjectivity. A transformer rejects opinion-style writing that isn't actually reporting.
  4. News-title shape. Another classifier rejects full sentences that don't look like a headline.
  5. Web presence. Searches the web, finds semantically similar headlines from trusted news domains, and asks an LLM whether they describe the same event.

If any check fails, the headline is flagged. The result page shows you exactly which gate it tripped on.

Limitations

  • The web-presence check defaults to DuckDuckGo (stable, no key). The Google backend is also wired up and uses Google's own tbm=nws classifier as a per-domain trust signal — the original 2023 design.
  • An LLM API key (Gemini or Anthropic) is required for the final gate; there's no local-only fallback by design.
  • This is a content-signal validator, not a fact-checker. Always cross-check with reputable sources before sharing.
